# 위험성평가 · TBM 웹앱 개발 로드맵 (v3)

> **목표**: 현장 작업자·관리감독자가 위험성평가표와 TBM 일지를 **중복 입력 0건**으로 작성하고, **SIF 사고사망 DB 2,574건과 기인물 기반 실시간 연계**로 정확성을 높이며, **지침서 붙임서식·TBM 양식과 100% 동일한 형태로 출력**하는 웹앱.

**기준 문서**:
- `5__위험성평가_지침서.pdf` — 붙임서식 3 (작업개요·위험성평가표·개선결과·결과확인)
- `TBM_양식.pdf` — TBM 일지 2면
- `12_1_위험성평가_양식r4.xlsx` — 7개 테이블 연동 알고리즘 원본
- **사용자 추가 프롬프트 (통합 DB 설계)** — SIF 연계 + 저장 기능
- **사용자 추가 프롬프트 (차트 시각화)** — 5종 차트 상세 설계

**핵심 원칙**: r4 엑셀의 7개 테이블 연동 로직을 그대로 이식하되, **SIF DB와 기인물을 공통 키로 연결**하여 유해위험요인·안전조치를 자동 추천하고, 모든 데이터는 **DB에 이력 저장**되어 재조회·재출력·통계 분석이 가능하도록 한다.

### 3-1. A7 테이블 — 기인물 체크 리스트 (58개 프리셋)

| 컬럼 | 설명 |
|------|------|
| A열 순번 | 1~58 |
| B열 기인물 | SIF 상위 58개 고정 목록 |
| C열 사망건수 | 참조용 (읽기 전용) |
| **D열 체크** | 🟡 드롭다운: `해당` / `미해당` |
| E열 중장비여부 | `중장비` / `굴착` / 빈값 |
| F열 고소감전 | `고소` / `감전` / 빈값 |

**역할**: 사용자가 D열에 체크한 결과가 H7(작업분류)·Z7(SIF 목록)·O7(투입중장비)·SIF 연계 패널 전부의 트리거.

### 3-2. H7 테이블 — 작업분류 10개 자동판정

**⚠️ v3에서 10개로 확장 (중량물작업 추가)**

| 순 | 구분 | 판정 유형 | 비고 |

**판정 공식 (엑셀 원본)**:

```
J = IF(OR(K="해당", L="해당"), "해당", "미해당")

L11 (고소): =IF(SUMPRODUCT(($F$8:$F$65="고소")*($D$8:$D$65="해당"))>=1, "해당", "미해당")
L12 (감전): =IF(SUMPRODUCT(($F$8:$F$65="감전")*($D$8:$D$65="해당"))>=1, "해당", "미해당")
L13 (굴착): =IF(SUMPRODUCT(($E$8:$E$65="굴착")*($D$8:$D$65="해당"))>=1, "해당", "미해당")
L14 (중장비):=IF(SUMPRODUCT(($E$8:$E$65="중장비")*($D$8:$D$65="해당"))>=1, "해당", "미해당")
```

```jsx
  { id: "굴착작업",   kind: "auto",   auto: h => h.some(x => x.checked && x.eGroup === "굴착") },
  { id: "중장비작업", kind: "auto",   auto: h => h.some(x => x.checked && x.eGroup === "중장비") },
  { id: "중량물작업", kind: "manual", note: "중량물작업계획서" },
  { id: "방사선작업", kind: "manual" },
  { id: "잠수작업",   kind: "manual" },
];

function classify(hazards, manualSet) {
  return workTypes.map(t => {
    const autoHit = t.kind === "auto" ? t.auto(hazards) : false;
    const manualHit = manualSet[t.id] === "해당";
    return { ...t, auto: autoHit, manual: manualHit,
             result: (autoHit || manualHit) ? "해당" : "미해당" };
  });
}
```

### 3-4. O7 테이블 — 작업개요 (노랑/회색셀 시각 구분)

**노랑셀 (직접입력)**: 회사명, 작성일자, 작성자, 작업기간, 작업장소, 중점관리사항, 필요한 보호구, 필요한 공기구

**회색셀 (자동반영)**:
- P8 = B2 (관리번호)
- P9 = B3 (작업명)
- P10 = B4 (작업내용)
- P13 = 작업분류 TEXTJOIN (`■ 일반작업 ■ 굴착작업`)

- R13 = 투입 중장비 (A7에서 체크된 중장비 기인물 + S13 수동추가 병합)

### 3-6. AB7 테이블 — 위험성평가 본체 DB (최대 300행)

| 열 | 내용 | 입력 방식 |
|----|------|---------|
| AB 공정번호 | 1-1, 1-2, 2-1... | 자동 부여 |
| AC 작업(공정)단계 | 🟡 U7 V열 값 드롭다운 |  |
| AD 유해위험요인 | 🟡 직접입력 **+ SIF 추천 팝업** |  |
| AE 안전조치사항 | 🟡 직접입력 **+ SIF 추천 팝업** |  |
| AF 가능성(전) | 🟡 1~5 |  |
| AG 중대성(전) | 🟡 1~5 |  |
| AH 위험도(전) | 🤖 = AF × AG |  |
| AI 가능성(후) | 🟡 1~5 |  |
| AJ 중대성(후) | 🟡 1~5 |  |
| AK 위험성(후) | 🤖 = AI × AJ |  |
| **🆕 AL TBM 체크** | ☑️ **체크박스** | **TBM 위험예지활동에 반영할 항목 선택** |

**TBM 체크 열 상세**:
- 기본값: **위험도(전) ≥ 9인 행은 자동 체크** (고위험 항목 자동 선별)
- 사용자가 체크박스를 수동으로 on/off 가능
- 체크된 행은 **TBM 일지 위험예지활동 표에 실시간 자동 반영**
- 체크 개수 제한: 없음 (원본 TBM 양식은 3행이지만, 동적으로 행 추가)
- 체크박스 상태는 위험성평가 DB에 `tbm_chk` 필드로 저장

**공정번호 자동 부여 로직**:

```jsx
function nextNo(rows, stageName) {
  const sameStage = rows.filter(r => r.stage === stageName);
  if (sameStage.length > 0) {
    const [major] = sameStage[0].no.split("-").map(Number);
    const maxMinor = Math.max(...sameStage.map(r => +r.no.split("-")[1]));
    return `${major}-${maxMinor + 1}`;
  }
  const majors = [...new Set(rows.map(r => +r.no.split("-")[0]))];
  return `${(majors.length ? Math.max(...majors) : 0) + 1}-1`;
}
```

### 3-7. AL7 테이블 — 공정별 위험성 분석 (자동 집계)

```
AL8: =IFERROR(INDEX(SORT(UNIQUE(FILTER(AC$8:AC$307, AC$8:AC$307<>""))), 1), "")
AM8: =IF(AL8="", "", COUNTIF(AC$8:AC$307, AL8))
AN8: =IF(AL8="", "", COUNTIFS(AC$8:AC$307, AL8, AH$8:AH$307, ">="&9))
AO8: =ROUND(AVERAGEIFS(AH$8:AH$307, AC$8:AC$307, AL8, AH$8:AH$307, "<>"), 1)

AQ8: =(AO8-AP8)/AO8
AR8: IF AO8>=9 "고위험" ELSE IF AO8>=6 "중위험" ELSE "저위험"
```

- 사용자가 필요에 따라 수동으로 on/off 조정
- 중위험(4~8) 항목도 체크 가능 (현장 판단)

# Part III. SIF DB 연계 (v3 신규)

## 4. SIF 연계 API 4종

### 4-1. API 1: 기인물 선택 시 SIF 정보 조회

```
GET /api/sif/기인물/{기인물명}
```

반환 데이터 (7종):

1. **기본정보**: 마스터 레코드 (순위·건수·구분)
2. **재해형태분포**: 떨어짐 125건(96.9%), 맞음 2건(1.6%)... → 도넛차트
3. **다빈도작업**: 작업명 Top 10 + 건수 → 수평막대
4. **공종분포**: 공종별 건수 → 파이차트
5. **사고사례**: 최근 5건 재해개요 → 카드 리스트
6. **개선대책**: Unique 목록 → 불릿 리스트

```sql
       m.총사고건수 AS SIF_총건수, m.다빈도_재해형태, m.위험도순위
FROM 위험성평가 w
JOIN SIF_기인물_마스터 m ON w.기인물 = m.기인물
WHERE w.관리번호 = 'RA-2025-0315'
ORDER BY m.위험도순위;

-- ② 고위험 공정 × SIF 사고사례 상세
SELECT w.작업공정단계, w.유해위험요인, w.위험성_전,
       s.재해형태, s.재해개요, s.위험성감소대책
FROM 위험성평가 w
JOIN SIF_사고사례 s ON w.기인물 = s.기인물
WHERE w.관리번호 = 'RA-2025-0315' AND w.위험성_전 >= 9
ORDER BY w.위험성_전 DESC, s.재해년월 DESC;

-- ③ 다빈도 기인물 Top 10 (전체 이력)
SELECT w.기인물,
       COUNT(DISTINCT w.관리번호) AS 사용_평가수,
       COUNT(*) AS 총_사용행수,
       m.총사고건수 AS SIF_사고건수, m.위험도순위
FROM 위험성평가 w
JOIN SIF_기인물_마스터 m ON w.기인물 = m.기인물
GROUP BY w.기인물, m.총사고건수, m.위험도순위
ORDER BY 사용_평가수 DESC
LIMIT 10;

-- ④ 월별 고위험 추이
SELECT DATE_TRUNC('month', j.작성일자::date) AS 월,
       COUNT(DISTINCT j.관리번호) AS 평가건수,
       SUM(CASE WHEN w.위험성_전 >= 9 THEN 1 ELSE 0 END) AS 고위험요인수,
       STRING_AGG(DISTINCT w.기인물, ', ') AS 관련기인물
FROM 작업개요 j JOIN 위험성평가 w ON j.관리번호 = w.관리번호
GROUP BY 1 ORDER BY 1 DESC;
```

# Part VIII. 검증 시나리오

## 18. E2E 테스트 케이스

1. **5단 연쇄 반응**: A7에서 "굴착기" 체크 → H7 "굴착작업"·"중장비작업" 자동 "해당" → O7 작업분류 자동 표시 → O7 투입중장비 자동 표시 → Z7·U7 드롭다운 자동 업데이트 → 섹션 8 SIF 패널에 "굴착기 104건 사고사례" 표시 → 위험 코멘트 자동 생성. **8단 연쇄**가 실시간으로 완료되는가?

2. **SIF 추천 팝업**: AD열에 포커스 → 해당 행의 기인물이 "비계"면 "비계 기반 상위 위험요인 Top 10" 팝업 → 2개 체크 후 "선택 적용" → AD열 자동 채움.

3. **위험도 실시간 갱신**: AF/AG 입력 → AH 즉시 계산 → AL 공정 분석 재집계 → AT 등급 분포 재계산 → 5종 차트 전부 리렌더링.

4. **저장·불러오기 왕복**: 관리번호 `RA-2025-0412-001`로 20행 입력 후 저장 → 새로고침 → 불러오기 모달에서 선택 → 모든 필드 복원 + 차트 재렌더링. SIF 연계 필드(선택 기인물 3개, SIF 총사고건수 349)도 복원.

5. **덮어쓰기 확인**: 같은 관리번호로 다시 저장 시 확인 모달 표시 → 기존 레코드 UPDATE + 위험성평가 행 전체 삭제 후 재삽입.

6. **PDF 일치도**: 인쇄 결과가 지침서 붙임서식 + TBM 양식의 표 구조·여백·폰트와 정확히 일치.

7. **참여자 축약**: 3명 → "김안전, 이안전, 박안전" / 12명 → "김안전, 이안전, 박안전, 최안전, 정안전 등 총 12명".

8. **고위험 공정 식별**: 평균 위험도 9 이상인 공정이 5개면 차트 5에 5개 카드 표시 + 각 카드에 상위 2개 유해위험요인.
